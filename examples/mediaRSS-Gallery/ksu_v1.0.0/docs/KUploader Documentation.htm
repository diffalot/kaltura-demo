Playlist documentation

Introduction:

The KUpload widget is a none graphical swf file that exposes an interface for uploading files to Kaltura network.

It exposes a set of configuration, API & event for JS applications.

The only direct user interaction is the browse action.

Due to security restrictions in Flash Player v10, any browse operation must be triggered as a result of user-interaction, therefore, user must click the swf itself in order to show the file dialogue box.

To overcome this security restriction, it's advised not to use the browse() API function, but rather place the swf above the real HTML GUI that the user sees and sets its wmode to "transparent", so that the actual "click" event is triggered from the flash and no API call is needed.

Public methods

Method signature

Description

addTags(tags:Array, startIndex:int, endIndex:int):void

Add tags to a range of FileVO objects.

tags  an Array that contains tags in each one if its items

startIndex & endIndex creates a range of FileVO to which the data should be added. To add tags to the first & second FileVO object tags, you call:

addTags(['doomed!', 'all doomed!'], 0, 1 )

The new tags will be appended to any existing tags

 

setTags(tags:Array, startIndex:int, endIndex:int):void

Set tags to a range of FileVO objects.

tags  an Array that contains tags in each one if its items

startIndex & endIndex creates a range of FileVO to which the data should be set. To set a single tag to the first & second FileVO object tags, you call:

setTags(['oh crap, dinosaurs!'], 0, 1 )

 

The new tags will replace any existing tags

 

setTitle(title:String, startIndex:int, endIndex:int):void

Set a titleto a range of FileVO objects.

title  A string that represents the media title

startIndex & endIndex creates a range of FileVO to which the data should be set. To set a title to the first FileVO object, you call:

setTitle(['I love fat books!'], 0, 0 )

 

removeFiles(startIndex:int, endIndex:int):void

Removes a range of file indicated by the startIndex & the endIndex

upload():void

Starts the upload operation if no limitation errors exists.

If the upload process was previously stopped, it will start form the file whose upload was cancelled

setMediaType(mediaType:String):void

Set the media type to show in the file dialogue.

The media types are defined in the UI conf file.

The common types are: video, audio, image.

addEntries():void

Executes the addentry http call to add any uploaded file

getFiles():Array

Returns an Array object that contains all FileVO objects

getTotalSize():uint

Returns the total size of all selected files

stopUploads():void

Stops any upload that is currently being performed.

getError():String

Returns the current error if any exists, otherwise, returns null.

Possible values:

·         fileSizeExceeds

·         totalSizeExceeds

·         numFilesExceeds

·         uploadError

getExceedingFilesIndices():Array

Returns an Array, whose values are indices of all files that exceeds the maxFileSize

getUploadedErrorIndices():Array

Returns an Array, whose values are indices of all files whose upload failed

setMaxUploads(value:uint):void

Sets the maximum number of uploads

setPartnerData(value:String):void

Sets (overwrites) the partner data that's sent on the addentry HTTP request

browse():void

[Deprecated] Show the file dialogue box.

It's strongly advised not to use this function but rather place the swf in a transparent mode above the HTML guid element that the user sees.

When the maxUploads is set to 1 file, the file dialogue box will only allow single file selection.

setGroupId(value:String):void

Sets the group id (overwrites any existing value)

setScreenName(value:String):void

Sets the screen name (overwrites any existing value)

setSiteUrl(value:String):void

Sets the site URL (overwrites any existing value)

setPermissions(value:String):void

Sets the permission level (overwrites any existing value)

4.    Errors  Exception that may be thrown by the flash player.

·         Limitations error  "Cannot upload, limitations exceeded. Please check for errors"

Thrown if the upload()  function is invoked while an error *exists.

·         Upload error - "Cannot add entries, some uploads failed. Either re-upload or remove the files"

Thrown if the addEntries() function is invoked while an error *exists.

 

* The error can be polled by calling getError()
